# Authentication API Documentation

## 1. SSO Token Authentication

### Endpoint
`POST https://e3engl0sk3.aitextbook.co.kr/api/v1/at/token`

### Description
Authenticates user with SSO token and obtains JWT access/refresh tokens

### Headers
```
Referer: https://e3engl0sk3.aitextbook.co.kr/sso?[SSO_PARAMS]
Content-Type: application/json
```

### SSO Query Parameters (in Referer URL)
| Parameter              | Example Value                            | Description                      |
| ---------------------- | ---------------------------------------- | -------------------------------- |
| access_token.token     | eyJjdHkiOiJKV1Q...                       | SSO token (encrypted JWT)        |
| access_token.access_id | 47d4de9263e300424d6fb4af8979cd18d52aebbc | Access identifier                |
| user_id                | 26af6255-e8af-57c7-8cd3-bd4981ba5ce3     | User UUID                        |
| user_status            | E                                        | User status (E=Enabled)          |
| user_type              | T                                        | User type (T=Teacher, S=Student) |
| api_domain             | https://api.aidtbook.kr                  | API service domain               |
| class_code             | 4V100000030_2025_00000000_1001           | Class code (30 chars)            |
| lecture_code           | 4V100000030_20251_00001001               | Lecture code (26 chars)          |
| partnerId              | fa1f5e94-7f48-563d-aa6f-a9c975f145f8     | Partner UUID                     |
| class_period           | 01                                       | Class period                     |
| cla_sel_yn             | N                                        | Class selection flag             |
| orgl_lecture_code      | 4V100000030_20251_00001001               | Original lecture code            |

### Request Body
```json
{
  "loginId": "4V100000030_20251_00001001-26af6255-e8af-57c7-8cd3-bd4981ba5ce3",
  "password": "4V100000030_20251_00001001-26af6255-e8af-57c7-8cd3-bd4981ba5ce3_!@12"
}
```

**Login ID Format:** `{lecture_code}-{user_id}`
**Password Format:** `{lecture_code}-{user_id}_!@12`

### Response Fields
| Field             | Description                    |
| ----------------- | ------------------------------ |
| timestamp         | Response timestamp (ISO 8601)  |
| code              | HTTP status code               |
| status            | Status message                 |
| data.accessToken  | JWT access token for API calls |
| data.refreshToken | JWT refresh token              |

### Token Details
- **Access Token**: Valid for 4 hours (expires at 1759137419)
- **Refresh Token**: Valid for 12 hours (expires at 1759166219)
- **Algorithm**: HS512 (HMAC SHA-512)

## Notes

1. **SSO Flow**: The Referer header contains the SSO context with encrypted JWT token
2. **Login Credentials**: Generated from lecture_code and user_id combination
3. **Token Usage**: The returned accessToken should be used in subsequent API calls
4. **Security**: In production, passwords should be properly hashed before storage
